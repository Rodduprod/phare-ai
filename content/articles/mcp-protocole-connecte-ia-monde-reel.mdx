## C'est quoi MCP ?

Le Model Context Protocol (MCP) est un standard ouvert qui définit comment un modèle de langage peut se connecter à des outils externes — bases de données, APIs, services SaaS, systèmes de fichiers. Pensez-y comme l'USB de l'IA : une interface universelle.

## Pourquoi c'est important

Jusqu'ici, connecter un LLM à un outil nécessitait du code custom pour chaque intégration. MCP standardise cette couche avec un protocole client-serveur simple :

- **Le client** (l'IA) découvre les outils disponibles
- **Le serveur** (votre outil) expose ses capacités via un schéma déclaratif
- **Le protocole** gère la négociation, l'authentification et le transport

Le résultat : un LLM compatible MCP peut utiliser n'importe quel outil MCP sans configuration spécifique.

## L'écosystème aujourd'hui

L'adoption est rapide. On trouve déjà des serveurs MCP pour :

- **Productivité** : Notion, Slack, Google Drive, Asana
- **Développement** : GitHub, Sentry, bases de données SQL
- **Business** : HubSpot, Salesforce, Amplitude
- **Contenu** : Webflow, Figma, Canva

## En pratique

Un agent IA avec MCP peut, sur une seule instruction :

1. Chercher un contact dans HubSpot
2. Vérifier ses dernières interactions dans Slack
3. Mettre à jour une tâche dans Notion
4. Envoyer un résumé par email

Sans écrire une seule ligne de code d'intégration.

## Ce qu'on surveille

La question ouverte est celle de la sécurité et du contrôle d'accès. Quand un agent peut toucher à tout, les garde-fous deviennent critiques. Les prochaines versions du protocole devraient adresser ça.
