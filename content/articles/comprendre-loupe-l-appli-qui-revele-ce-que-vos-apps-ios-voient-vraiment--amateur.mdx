# Comprendre Loupe, l’appli qui révèle ce que vos apps iOS voient vraiment

On adore les applis qui promettent de "simplifier votre vie", "révolutionner votre productivité" ou "vous comprendre mieux que votre mère". Mais on parle rarement de ce qu’elles voient *vraiment* sous le capot. Loupe, une appli iOS open source développée par les chercheurs de Mysk, change la donne en affichant crûment ce à quoi vos apps natives ont accès. Pas de jargon marketing, pas de "confiance aveugle" : juste les faits, bruts de décoffrage.

## Contexte et enjeux : pourquoi Loupe fait mal

iOS a la réputation d’être un écosystème verrouillé, sécurisé, presque aseptisé. Pourtant, même dans ce jardin clos, les apps peuvent accéder à une quantité surprenante de données. Pas toujours avec de mauvaises intentions, mais souvent sans que l’utilisateur en ait conscience.

Loupe, c’est un peu comme si on vous tendait une lampe torche dans une pièce sombre : soudain, vous voyez les coins que vous ignoriez. L’appli scanne les permissions demandées par les autres apps installées et affiche ce qu’elles peuvent *vraiment* collecter. Localisation, contacts, photos, microphone, mais aussi des données moins évidentes comme l’historique du presse-papiers, les réseaux Wi-Fi à proximité, ou même l’activité des autres apps en arrière-plan.

Et le pire ? Beaucoup de ces accès sont légitimes. Une appli de fitness a besoin de votre localisation pour tracer vos runs. Une appli de messagerie a besoin de vos contacts pour vous suggérer des amis. Mais quand une appli de lampe torche demande l’accès à vos photos, on peut se poser des questions. Loupe ne juge pas, elle expose. À vous de tirer les conclusions.

D’après les tests menés par [Apple et l’IA : ce que les pros tech doivent comprendre derrière les annonces](/articles/apple-et-l-ia-ce-que-votre-iphone-va-vraiment-faire-spoiler-pas-la-vaisselle--amateur), même les utilisateurs avertis sous-estiment souvent l’étendue des permissions accordées. Et avec l’arrivée de l’IA on-device, ces questions deviennent encore plus critiques.

## Fonctionnement : comment Loupe fait son travail

Loupe ne fait pas de magie. Elle exploite simplement les APIs publiques d’iOS pour lister les permissions déclarées par chaque appli. Voici ce qu’elle analyse, en gros :

**Les permissions classiques** : localisation, appareil photo, microphone, contacts, calendrier, rappels, photos, Bluetooth, santé, etc. Rien de bien nouveau, mais voir la liste complète pour une appli donnée peut faire froid dans le dos.

**Les permissions moins connues** :
- **Accès au presse-papiers** : certaines apps lisent ce que vous copiez-collez, même si vous ne les utilisez pas. Pratique pour un gestionnaire de mots de passe, moins pour un jeu mobile.
- **Réseaux Wi-Fi et Bluetooth** : savoir quels réseaux sont à proximité peut révéler votre position, même sans GPS activé.
- **Activité en arrière-plan** : certaines apps peuvent détecter quand vous utilisez d’autres applis. Utile pour un tracker de temps d’écran, inquiétant pour une appli de shopping.

**Les données système** : version d’iOS, modèle de l’appareil, espace de stockage disponible, etc. Des infos qui semblent anodines, mais qui, combinées, peuvent créer un profil assez précis.

L’appli ne nécessite aucune permission particulière pour fonctionner : elle se contente de lire les métadonnées des autres apps, accessibles via les APIs d’iOS. Pas de root, pas de jailbreak, juste une lecture transparente de ce que le système autorise déjà.

Le code est open source, disponible sur [GitHub](https://github.com/mysk-research/loupe). Vous pouvez donc vérifier par vous-même que Loupe ne fait pas plus qu’afficher ces informations. Et si vous êtes paranoïaque (ce qui est sain en matière de privacy), vous pouvez même compiler l’appli vous-même.

Pour un pro tech, Loupe n’est pas juste un outil de parano. C’est un moyen de **valider la conformité RGPD** de vos propres applis ou de celles que vous utilisez en entreprise.

Avant de déployer une nouvelle appli en interne, passez-la à la loupe (sans jeu de mots). Vous serez surpris de voir combien d’apps "innocentes" demandent des permissions inutiles. Une appli de notes qui veut accéder à votre localisation ? Un outil de productivité qui lit vos contacts ? Ces demandes peuvent être légitimes, mais elles doivent être justifiées et documentées.

### Sensibilisation des équipes
Montrez Loupe à vos collègues non-tech. Rien de tel qu’une démo en direct pour leur faire comprendre pourquoi il ne faut pas installer n’importe quelle appli sur un appareil professionnel. Et si votre entreprise a une politique BYOD (Bring Your Own Device), Loupe peut devenir un outil de formation indispensable.

### Développement d’apps respectueuses
Si vous développez des applis iOS, Loupe peut vous aider à **minimiser les permissions demandées**. Moins de permissions = moins de risques de fuite de données, moins de questions des utilisateurs, et une meilleure réputation en matière de privacy. ## APIs et intégration : peut-on automatiser ça ?

Loupe est une appli grand public, mais son approche peut inspirer des outils plus avancés pour les entreprises. Voici ce que vous pourriez reproduire en interne :

### Utilisation des APIs iOS
Les APIs utilisées par Loupe sont documentées par Apple. Vous pouvez donc développer votre propre outil pour scanner les permissions des apps sur une flotte d’appareils. Voici les principales APIs à explorer :

- **`LSApplicationWorkspace`** (dépréciée mais encore utilisée) ou **`UIApplication`** pour lister les apps installées.
- **`NSFileManager`** pour accéder aux plists des apps (où sont déclarées les permissions).
- **`CLLocationManager`, `AVFoundation`, `Contacts`, etc.** pour vérifier les permissions accordées.

Un exemple simple en Swift pour lister les apps et leurs permissions :

```swift
import UIKit

func listAppsWithPermissions() {
    let fileManager = FileManager.default
    let appsURL = fileManager.urls(for: .applicationDirectory, in: .userDomainMask).first!

    do {
        let apps = try fileManager.contentsOfDirectory(at: appsURL, includingPropertiesForKeys: nil)
        for app in apps {
            let appPath = app.path
            if let infoPlist = NSDictionary(contentsOfFile: "\(appPath)/Info.plist") {
                let permissions = infoPlist["NSLocationWhenInUseUsageDescription"] as? String
                // Ajoutez d'autres clés comme NSCameraUsageDescription, NSMicrophoneUsageDescription, etc.
                print("App: \(app.lastPathComponent), Permissions: \(permissions ?? "Aucune")")
            }
        }
    } catch {
        print("Erreur: \(error)")
    }
}
```

### Intégration dans un MDM
Si votre entreprise utilise un Mobile Device Management (MDM) comme Jamf, Mosyle ou Kandji, vous pouvez intégrer ce type de scan dans vos politiques de sécurité. Certains MDM permettent déjà de lister les apps installées et leurs permissions, mais un outil dédié comme Loupe peut offrir une granularité supplémentaire.

## FAQ

**[Loupe peut-elle désactiver les permissions des apps ?]**
Non, Loupe est un outil de visualisation uniquement. Pour désactiver des permissions, il faut passer par les réglages iOS (Paramètres > Confidentialité et sécurité). Mais Loupe vous indique exactement où chercher.

**[Loupe fonctionne-t-elle sur Android ?]**
Non, Loupe est spécifique à iOS. Mais des outils similaires existent pour Android, comme [Exodus Privacy](https://exodus-privacy.eu.org/) ou [Permission Explorer](https://play.google.com/store/apps/details?id=com.ramdroid.permissionexplorer).

**[Est-ce que Loupe peut détecter des malwares ?]**
Non, Loupe ne scanne pas le comportement des apps, seulement leurs permissions déclarées. Un malware pourrait très bien demander peu de permissions pour passer inaperçu. Pour une détection active, il faut des outils de sécurité dédiés.
