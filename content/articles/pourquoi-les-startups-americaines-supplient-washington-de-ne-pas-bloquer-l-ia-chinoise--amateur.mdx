## Fonctionnement : pourquoi les modèles open weights changent la donne

Un modèle open weights, c’est un peu comme une voiture dont on vous donne le plan détaillé et toutes les pièces détachées. Vous pouvez l’utiliser tel quel, ou le modifier pour l’adapter à vos besoins. Contrairement aux modèles fermés (comme ceux de OpenAI ou Anthropic), où vous n’avez accès qu’à l’interface de conduite, sans savoir ce qu’il y a sous le capot.

**Architecture typique d’un modèle open weights chinois :**
- **Prétraînement massif** : des milliards de tokens ingurgités, souvent sur des datasets multilingues (le chinois, mais aussi l’anglais, le français, etc.).
- **Optimisation pour le low-cost** : ces modèles sont conçus pour tourner sur des infrastructures moins puissantes que celles de Nvidia. Résultat, ils consomment moins d’énergie et coûtent moins cher à déployer.
- **Fine-tuning accessible** : avec les poids ouverts, vous pouvez affiner le modèle sur vos propres données, sans dépendre d’une API propriétaire.

Prenons l’exemple de **Qwen 3**, le dernier né d’Alibaba. Selon les benchmarks, il rivalise avec des modèles comme Llama 3 de Meta, mais avec une empreinte carbone et un coût d’inférence bien inférieurs. Et comme il est open weights, une startup peut le télécharger, le modifier, et le déployer sur ses propres serveurs sans payer de licence.

C’est là que le bât blesse pour les États-Unis. Si Washington interdit l’accès à ces modèles, les startups américaines devront soit :
1. **Payer cher** pour utiliser des modèles fermés (et souvent moins performants pour leur cas d’usage spécifique).
2. **Se tourner vers des alternatives européennes**, comme celles de Mistral AI. Mais même là, les coûts et les contraintes techniques restent élevés.

## APIs disponibles : comment les utiliser sans se faire griller

Si vous voulez tester ces modèles sans vous prendre la tête, voici quelques options :

### 1. **Hugging Face**
La plateforme propose une myriade de modèles open weights, y compris chinois. Vous pouvez les télécharger, les tester en local, ou les déployer via leur API.
Exemple de code pour charger Qwen 3 :

```python
from transformers import AutoModelForCausalLM, AutoTokenizer

model_name = "Qwen/Qwen3-8B"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(model_name, device_map="auto")
```

### 2. **Together AI**
Cette plateforme permet d’accéder à des modèles open weights via une API simple, avec des options de fine-tuning.
Exemple d’appel API :

```bash
curl -X POST https://api.together.xyz/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "Qwen/Qwen3-8B", "messages": [{"role": "user", "content": "Explique-moi l'IA en 3 mots."}]}'
```

### 3. **Ollama**
Pour ceux qui veulent tout faire en local, Ollama permet de déployer des modèles open weights sur votre machine.
Installation et utilisation :

```bash
ollama pull qwen3:8b
ollama run qwen3:8b "Pourquoi l'IA chinoise fait-elle peur aux États-Unis ?"
```

## FAQ

**[Pourquoi les modèles chinois open weights sont-ils moins chers ?]**
Ils sont optimisés pour tourner sur des infrastructures moins coûteuses (puces chinoises, cloud local) et bénéficient de subventions gouvernementales. De plus, leur architecture est souvent conçue pour être légère et économe en ressources.

**[Peut-on vraiment faire confiance à un modèle chinois open weights ?]**
C’est une question de risque acceptable. Techniquement, oui : vous pouvez auditer le code et les données. Mais il faut être conscient des potentiels backdoors ou biais introduits volontairement ou non. Comme pour tout outil open source, la vigilance est de mise.

**[Quelles alternatives si Washington bloque l’accès à ces modèles ?]**
Les startups pourraient se tourner vers des modèles européens (Mistral AI, Aleph Alpha) ou développer leurs propres architectures. Mais le coût et le temps de développement seraient bien supérieurs, ce qui freinerait l’innovation.
