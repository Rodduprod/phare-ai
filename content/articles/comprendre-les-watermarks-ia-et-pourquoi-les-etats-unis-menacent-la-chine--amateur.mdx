# Comprendre les watermarks IA et pourquoi les États-Unis menacent la Chine

On ne va pas se mentir : les *watermarks* dans les modèles de langage, c’est un peu comme les tatouages sur un espion. Invisibles à l’œil nu, mais cruciaux quand il s’agit de prouver d’où vient l’information. Et aujourd’hui, ces petits marqueurs numériques sont au cœur d’une tension entre Washington et Pékin.

D’après *TradingKey*, le secrétaire au Trésor américain aurait récemment menacé de sanctions contre les modèles chinois comme **Kimi K3** de Moonshot AI, après avoir détecté des *watermarks* typiques des LLMs américains dans leurs sorties. Une révélation qui sent le scandale industriel… ou la guerre froide 2.0.

## Contexte et enjeux : pourquoi ça chauffe entre les USA et la Chine

La Chine a un problème : ses modèles d’IA les plus performants, comme [Kimi K3](https://lelabo.ai/articles/comprendre-kimi-k3-l-ia-chinoise-qui-veut-tout-faire-sans-faire-de-vagues--amateur) ou [Qwen d’Alibaba](https://lelabo.ai/articles/alibaba-et-qwen-le-geant-chinois-qui-ne-lache-rien-sur-l-ia--amateur), semblent s’être entraînés sur des données américaines. Pas illégal en soi, sauf que les États-Unis commencent à voir d’un mauvais œil que leurs propres *watermarks* — ces signatures numériques intégrées dans les sorties des LLMs pour tracer leur origine — réapparaissent dans des modèles étrangers.

Pourquoi c’est un problème ? Parce que ces *watermarks* peuvent révéler deux choses :
1. **L’utilisation de données américaines** pour l’entraînement, ce qui pose question sur la souveraineté des modèles chinois.
2. **Une possible violation des restrictions d’exportation** si les modèles chinois intègrent des composants ou des poids issus de modèles américains (comme ceux de Nvidia, par exemple).

Franchement, c’est un peu comme si un restaurant chinois servait des burgers McDonald’s en prétendant que c’est de la cuisine traditionnelle. Ça passe mal.

## Fonctionnement : comment ça marche, un watermark dans un LLM ?

Un *watermark* dans un modèle de langage, c’est une technique qui permet d’identifier de manière probabiliste si un texte a été généré par une IA spécifique. Contrairement à un filigrane classique (comme ceux des billets de banque), ici, on ne parle pas d’une image cachée, mais d’un **pattern statistique** dans les sorties du modèle.

### Les méthodes les plus courantes
- **Watermarking par distribution de tokens** : le modèle est légèrement biaisé pour favoriser certains mots ou séquences de mots selon une clé secrète. Résultat : un texte généré par ce modèle aura une "signature" détectable.
- **Watermarking par hachage** : on injecte dans le prompt ou dans les poids du modèle une séquence de tokens qui, une fois hachée, donne une signature unique.
- **Watermarking par perturbation** : on ajoute un bruit contrôlé dans les sorties, imperceptible pour un humain mais détectable par un algorithme.

L’avantage ? Ces méthodes sont **résistantes aux modifications mineures** (comme la réécriture ou la traduction). L’inconvénient ? Elles peuvent être contournées si on connaît la clé ou si on utilise des techniques d’*adversarial attacks*.

### Exemple concret avec Kimi K3
D’après les rapports, Kimi K3 aurait été entraîné sur des données incluant des sorties de LLMs américains (comme ceux de Meta ou Mistral). Résultat : ses propres sorties conservent des traces de ces *watermarks*, comme une sorte d’ADN numérique.

C’est un peu comme si vous copiiez un devoir en changeant deux mots : le prof voit tout de suite que c’est du plagiat.

## Cas d’usage business : à quoi servent les watermarks en entreprise ?

Pour les pros tech, les *watermarks* ne sont pas qu’un outil géopolitique. Ils ont des applications très concrètes :

### 1. **Traçabilité des contenus générés**
Vous utilisez un LLM pour produire des rapports, du code ou du contenu marketing ? Les *watermarks* permettent de **prouver l’origine** de ce contenu en cas de litige (propriété intellectuelle, plagiat, etc.).

### 2. **Contrôle des fuites de données**
Si un employé exfiltre des données sensibles via un chatbot interne, les *watermarks* peuvent aider à **remonter jusqu’à la source** de la fuite.

### 3. **Conformité réglementaire**
Avec des lois comme l’**AI Act** en Europe ou les restrictions américaines sur l’export de technologies IA, les *watermarks* deviennent un outil de **conformité**. Ils permettent de certifier que vos modèles respectent les règles (ou pas).

### 4. **Lutte contre la désinformation**
Les *watermarks* peuvent aider à identifier les **deepfakes textuels** ou les contenus générés par des IA malveillantes. Un peu comme un [système de détection de deepfakes](https://lelabo.ai/articles/bernie-sanders-piege-par-une-fausse-video-ia-le-flop-qui-fait-rire--confirme), mais pour le texte.

## APIs disponibles : comment implémenter des watermarks dans vos projets

Si vous voulez jouer avec les *watermarks* dans vos propres modèles, voici quelques outils et bibliothèques open source à connaître :

### 1. **Watermarking pour LLMs (Hugging Face)**
La bibliothèque [`transformers`](https://huggingface.co/docs/transformers/main_classes/text_generation) de Hugging Face permet d’intégrer des *watermarks* via des *logits processors*. Exemple en Python :

```python
from transformers import AutoModelForCausalLM, AutoTokenizer, LogitsProcessor

class WatermarkLogitsProcessor(LogitsProcessor):
    def __init__(self, watermark_key):
        self.watermark_key = watermark_key

    def __call__(self, input_ids, scores):
        # Appliquer un biais léger sur certains tokens selon la clé
        for i, token_id in enumerate(self.watermark_key):
            if i < len(scores[0]):
                scores[0][token_id] += 0.1  # Biais minimal
        return scores

model = AutoModelForCausalLM.from_pretrained("mistralai/Mistral-7B")
tokenizer = AutoTokenizer.from_pretrained("mistralai/Mistral-7B")
watermark_processor = WatermarkLogitsProcessor(watermark_key=[123, 456, 789])  # Exemple de clé

# Génération avec watermark
output = model.generate(
    input_ids=tokenizer.encode("Explique-moi les watermarks IA", return_tensors="pt"),
    logits_processor=watermark_processor,
    max_length=100
)
print(tokenizer.decode(output[0]))
```

### 2. **Watermarking avec la bibliothèque `watermark`**
Le projet [`watermark`](https://github.com/ShieldMnt/invisible-watermark) (par ShieldMnt) propose une implémentation simple pour ajouter des *watermarks* invisibles dans du texte ou des images.

### 3. **Détection de watermarks**
Pour détecter si un texte contient un *watermark*, on peut utiliser des outils comme :
- **`detect-watermark`** (bibliothèque Python)
- **Les APIs de détection** proposées par des startups comme **Arize AI** ou **WhyLabs**.

## ROI et impact sur les équipes : est-ce que ça vaut le coup ?

### **Coûts**
- **Implémentation** : Ajouter un *watermark* à un LLM existant est **peu coûteux** (quelques heures de dev pour un ingénieur ML).
- **Détection** : Plus cher, car il faut entraîner des modèles de détection ou utiliser des APIs tierces.
- **Performance** : Un *watermark* bien conçu a un **impact minimal** sur les performances du modèle (moins de 1% de perte en précision).

### **Bénéfices**
- **Sécurité** : Réduction des risques de fuite de données ou de plagiat.
- **Conformité** : Respect des réglementations (RGPD, AI Act, etc.).
- **Confiance** : Preuve de transparence pour les clients ou partenaires.

### **Risques**
- **Contournement** : Les *watermarks* ne sont pas infaillibles. Un attaquant motivé peut les supprimer avec des techniques d’*adversarial attacks*.
- **Faux positifs** : Un *watermark* mal conçu peut déclencher des alertes sur du contenu légitime.
- **Complexité** : Gérer des *watermarks* à grande échelle (plusieurs modèles, plusieurs versions) peut devenir un casse-tête.

### **Impact sur les équipes**
- **Pour les data scientists** : Il faut intégrer les *watermarks* dans le pipeline d’entraînement et de déploiement.
- **Pour les juristes** : Les *watermarks* deviennent un outil clé pour la **propriété intellectuelle** et la conformité.
- **Pour les ops** : Il faut surveiller les sorties des modèles pour détecter d’éventuelles anomalies.

## FAQ

**[C’est quoi exactement un watermark dans un LLM ?]**
Un watermark est une signature numérique intégrée dans les sorties d’un modèle de langage, permettant de tracer son origine. Contrairement à un filigrane visible, il s’agit d’un pattern statistique ou d’un biais appliqué aux probabilités de génération des tokens.

**[Pourquoi les États-Unis s’inquiètent des watermarks dans Kimi K3 ?]**
Parce que la détection de watermarks américains dans les sorties de Kimi K3 suggère que le modèle chinois a été entraîné sur des données ou des sorties de LLMs américains, ce qui pourrait violer les restrictions d’exportation ou poser des questions de souveraineté technologique.

**[Est-ce que les watermarks peuvent être supprimés ?]**
Oui, mais c’est difficile. Des techniques comme la réécriture, la traduction ou les attaques adversariales peuvent les atténuer, mais un watermark bien conçu reste détectable dans la plupart des cas.
