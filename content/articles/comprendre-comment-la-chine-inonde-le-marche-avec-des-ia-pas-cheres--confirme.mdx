# Comprendre l’architecture du modèle IA chinois low-cost qui talonne OpenAI

On ne va pas se mentir : quand un modèle chinois débarque en fanfare avec des performances proches de GPT-4 ou Claude 3 pour trois fois rien, ça fait tiquer. Surtout quand on sait que les coûts d’entraînement d’un LLM de cette trempe peuvent atteindre des sommets dignes de l’Everest. Pourtant, selon *News24*, c’est bien ce que propose un nouveau venu sur le marché, avec une approche qui rappelle étrangement la stratégie d’Alibaba avec [Qwen 3](https://lelabo.ai/articles/alibaba-et-qwen-le-geant-chinois-qui-ne-lache-rien-sur-l-ia--confirme). Alors, miracle technique ou coup marketing ? Spoiler : un peu des deux.

## Les fondements techniques : comment faire plus avec moins

L’astuce ? **La distillation de modèles.** Pas celle de votre grand-oncle dans son garage, non. Ici, on parle de *knowledge distillation*, une technique qui consiste à entraîner un petit modèle (le *student*) pour qu’il imite les comportements d’un modèle géant (le *teacher*). Le résultat : un LLM de taille modeste, mais qui se comporte comme un modèle bien plus gros.

Le modèle en question utilise une variante avancée de cette méthode, avec quelques twists :
- **Distillation multi-teacher** : au lieu de s’inspirer d’un seul modèle lourd, il agrège les connaissances de plusieurs *teachers* (GPT-4, Claude 3, mais aussi des modèles open source comme Llama 3). Résultat, une généralisation meilleure que la moyenne.
- **Fine-tuning ciblé** : les données d’entraînement sont optimisées pour les tâches où les modèles occidentaux excellent (raisonnement, code, maths), mais avec un focus sur les *edge cases* souvent négligés (langues asiatiques, contextes culturels locaux).
- **Quantification agressive** : le modèle tourne en **INT4** ou **INT8** sans perte significative de performance, grâce à des techniques comme *GPTQ* ou *AWQ*. Franchement, c’est impressionnant, mais bon, on parle quand même d’une compression qui frôle l’overfitting.

Et puis il y a l’optimisation matérielle. Les Chinois misent sur des **accélérateurs maison** (comme ceux de [Meituan](https://lelabo.ai/articles/comprendre-le-geant-ia-de-meituan-et-ses-puces-chinoises-maison--confirme)) et des frameworks légers (ex : *vLLM* pour l’inference). Résultat, un coût par token divisé par 5 à 10 par rapport à un déploiement classique sur A100.

## Implémentation : sous le capot, ça ressemble à quoi ?

Passons aux choses sérieuses. Voici ce qu’on sait de l’architecture :

Le modèle repose sur une **variante de Transformer** avec :
- **Attention groupée** (*Grouped-Query Attention*) pour réduire la mémoire utilisée sans sacrifier la qualité. Une astuce déjà vue chez Mistral AI, mais poussée plus loin ici.
- **MoE léger** : pas de *Mixture of Experts* à la GPT-4, mais une version allégée où seulement 2 experts sont activés par couche. Assez pour gagner en efficacité, pas assez pour exploser le budget.
- **Cache KV optimisé** : les clés et valeurs de l’attention sont compressées en temps réel, ce qui permet de gérer des contextes plus longs sans OOM.

Côté code, voici un extrait simplifié de l’implémentation de l’attention groupée (en PyTorch) :

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class GroupedQueryAttention(nn.Module):
    def __init__(self, embed_dim, num_heads, num_groups):
        super().__init__()
        self.embed_dim = embed_dim
        self.num_heads = num_heads
        self.num_groups = num_groups
        self.head_dim = embed_dim // num_heads

        assert num_heads % num_groups == 0, "num_heads must be divisible by num_groups"
        self.group_size = num_heads // num_groups

        self.q_proj = nn.Linear(embed_dim, embed_dim)
        self.k_proj = nn.Linear(embed_dim, embed_dim // num_groups * num_groups)
        self.v_proj = nn.Linear(embed_dim, embed_dim // num_groups * num_groups)
        self.out_proj = nn.Linear(embed_dim, embed_dim)

    def forward(self, x):
        batch_size, seq_len, _ = x.shape

        # Projections
        q = self.q_proj(x)  # [batch, seq_len, embed_dim]
        k = self.k_proj(x)  # [batch, seq_len, embed_dim//num_groups * num_groups]
        v = self.v_proj(x)  # [batch, seq_len, embed_dim//num_groups * num_groups]

        # Reshape pour GQA
        q = q.view(batch_size, seq_len, self.num_groups, self.group_size, self.head_dim).transpose(2, 3)
        k = k.view(batch_size, seq_len, self.num_groups, self.group_size, self.head_dim).transpose(2, 3)
        v = v.view(batch_size, seq_len, self.num_groups, self.group_size, self.head_dim).transpose(2, 3)

        # Attention scores
        attn_scores = torch.matmul(q, k.transpose(-2, -1)) / (self.head_dim ** 0.5)
        attn_probs = F.softmax(attn_scores, dim=-1)

        # Output
        output = torch.matmul(attn_probs, v)
        output = output.transpose(2, 3).contiguous().view(batch_size, seq_len, -1)
        return self.out_proj(output)
```

Bien sûr, ce n’est qu’un morceau du puzzle. Le vrai travail réside dans l’optimisation des *kernels* CUDA et l’intégration avec les accélérateurs matériels locaux.

## Benchmarks : où il brille, où il déçoit

Les annonces marketing parlent de performances "proches de GPT-4". En réalité, c’est plus nuancé.

**Ce qui marche bien :**
- **Compréhension multilingue** : le modèle surpasse la plupart des LLM occidentaux sur les tâches en mandarin, cantonais, et même en vietnamien. Preuve que le fine-tuning culturel paie.
- **Efficacité énergétique** : en INF4 (4 bits), il consomme **30% d’énergie en moins** qu’un Llama 3 en INT8, pour des performances similaires sur la plupart des benchmarks.
- **Latence** : grâce à la quantification et à l’attention groupée, il répond **2x plus vite** qu’un Claude 3 Haiku sur des requêtes courtes.

**Ce qui coince :**
- **Raisonnement complexe** : sur des tâches comme *GPQA* (raisonnement scientifique avancé) ou *MMLU-Pro*, il reste **10-15% derrière** GPT-4o. La distillation a ses limites.
- **Créativité** : les réponses sont souvent **plus prévisibles**, avec moins de diversité. Un effet secondaire classique de la compression de modèles.
- **Stabilité** : en mode *chat long*, il a tendance à **dériver** après 20-30 tours de conversation, un problème récurrent avec les modèles distillés.

## Limitations : les compromis du low-cost

**1. La dépendance aux modèles teachers**
Si les *teachers* (GPT-4, Claude 3) changent radicalement de comportement, le modèle distillé peut se retrouver **dépassé du jour au lendemain**. C’est le risque de la distillation : on hérite des forces… et des faiblesses de ses mentors.

**2. La généralisation limitée**
Le modèle excelle sur les tâches pour lesquelles il a été spécifiquement optimisé (code, maths, langues asiatiques), mais **décroche sur les cas de niche**. Par exemple, il est **mauvais en droit français** ou en analyse de données médicales occidentales.

**3. L’opacité des données d’entraînement**
Contrairement à Mistral ou Llama, il n’y a **aucune transparence** sur les datasets utilisés. On ne sait pas si des données protégées (brevets, œuvres sous copyright) ont été incluses. Un risque juridique non négligeable pour les entreprises qui voudraient l’adopter.

**4. L’écosystème limité**
Pas de *plugin system* à la ChatGPT, pas d’intégration native avec les outils occidentaux (GitHub, Notion, etc.). Si vous voulez l’utiliser en production, préparez-vous à **tout coder maison**.

## Recherche et évolutions futures : vers une IA low-cost dominante ?

La Chine ne compte pas s’arrêter là. Voici ce qui se profile :

**1. Des modèles encore plus légers**
Les chercheurs travaillent sur des **LLM en INT3** (3 bits !) avec des techniques comme *BitNet* ou *Ternary Weight Networks*. L’objectif : faire tourner un modèle de 70B de paramètres sur un **Raspberry Pi 5**. On en est loin, mais les progrès sont rapides.

**2. L’intégration avec le hardware local**
Des puces comme le *Huawei Ascend 910B* ou le *Moore Threads MUSA* sont optimisées pour ces modèles. Résultat : des **coûts d’inference divisés par 20** par rapport à un déploiement sur GPU NVIDIA.

**3. La guerre des benchmarks**
Les Chinois poussent pour que des benchmarks **locaux** (comme *C-Eval* pour le chinois ou *CMMLU* pour le multilingue) deviennent la référence. Une stratégie intelligente : si les tests sont conçus pour leurs modèles, ils gagneront mécaniquement.

**4. L’open source comme arme**
Contrairement à OpenAI ou Anthropic, la Chine mise sur l’**open source agressif**. Des modèles comme *Qwen* ou *InternLM* sont déjà disponibles en licence permissive. Le but ? **Inonder le marché** pour que les développeurs adoptent leurs standards.
