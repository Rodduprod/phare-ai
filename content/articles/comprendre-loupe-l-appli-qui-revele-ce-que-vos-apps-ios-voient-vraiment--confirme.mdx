# Comprendre Loupe, l’appli qui révèle ce que vos apps iOS voient vraiment

On ne va pas se mentir : les apps iOS sont des espions en costume-cravate. Elles accèdent à vos contacts, votre localisation, vos photos, et même à des capteurs que vous ne saviez pas exister. **Loupe**, une appli open source développée par les chercheurs de Mysk, lève le voile sur ce bazar. Pas de marketing tape-à-l’œil, pas de promesses creuses. Juste du code qui montre, noir sur blanc, ce que vos applications font dans votre dos.

## Les fondements techniques : comment iOS gère (mal) vos données

iOS a un système de permissions. En théorie, c’est simple : une app demande, vous acceptez ou refusez. En pratique, c’est un peu comme un contrat de téléphonie mobile : vous signez sans lire, et trois mois plus tard, vous vous demandez pourquoi votre forfait coûte 200€.

Les permissions sur iOS sont gérées via **Info.plist**, un fichier de configuration qui déclare ce dont l’app a besoin. Problème : certaines permissions sont *silencieuses*. Par exemple, une app peut accéder à votre **pasteboard** (presse-papiers) sans vous prévenir. Ou pire, utiliser des **API privées** pour récupérer des données comme l’**identifiant publicitaire** (IDFA), l’**adresse MAC**, ou même l’**état de la batterie**.

Loupe exploite les **API publiques d’iOS** pour lister :
- Les permissions déclarées dans l’Info.plist
- Les **entitlements** (droits étendus) accordés à l’app
- Les **appels système** suspects (comme les accès aux capteurs de mouvement ou au Bluetooth)

Le tout en temps réel, avec un historique pour voir qui a fait quoi, quand.

```swift
// Exemple de code pour lister les permissions dans Loupe
let permissions = Bundle.main.infoDictionary?["UIRequiredDeviceCapabilities"] as? [String]
let entitlements = SecCodeCopySigningInformation(kSecCSDefaultFlags).takeRetainedValue() as? [String: Any]
```

## Implémentation : sous le capot de Loupe

Loupe est écrite en **Swift**, avec une architecture modulaire qui sépare :
- **Le moteur de détection** : analyse statique des binaires et dynamique des appels système
- **L’interface utilisateur** : un tableau de bord minimaliste qui affiche les alertes en temps réel
- **Le module de logging** : stocke les événements pour analyse ultérieure

L’appli utilise **DTrace** et **dtrace(1)** pour surveiller les appels système. Oui, c’est un outil de debugging vieux comme le monde, mais c’est toujours le plus efficace pour voir ce que font vraiment les processus.


## Limitations : ce que Loupe ne peut pas faire

Loupe a des limites, et c’est normal :
- **Pas de détection des accès réseau** : elle ne voit pas *ce que* l’app envoie, seulement *qu’elle envoie quelque chose*.
- **Jailbreak requis pour le deep monitoring** : sans root, certaines API restent invisibles.
- **Pas de protection active** : Loupe alerte, mais ne bloque rien. À vous de désinstaller les apps indésirables.

Et puis, il y a le problème récurrent des apps qui **contournent les permissions**. Par exemple, une app peut demander l’accès à la caméra pour un "filtre photo", puis utiliser cette permission pour scanner votre environnement en arrière-plan. Loupe détecte l’accès, mais pas l’intention malveillante.

## FAQ

**[Loupe fonctionne-t-elle sans jailbreak ?]**
Oui, mais avec des limitations. Sans jailbreak, Loupe peut lister les permissions déclarées et certains accès, mais pas surveiller les appels système en temps réel.

**[Est-ce que Loupe peut bloquer les accès indésirables ?]**
Non. Loupe est un outil de détection, pas de protection. Elle vous alerte, mais c’est à vous d’agir (désinstaller l’app, refuser les permissions, etc.).

**[Pourquoi Loupe n’est pas sur l’App Store ?]**
Apple n’autorise pas les apps qui analysent le comportement des autres apps. Loupe doit donc être installée via **sideloading** (AltStore, Sideloadly) ou un jailbreak.
