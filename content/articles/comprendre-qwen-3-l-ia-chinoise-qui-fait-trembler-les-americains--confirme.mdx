## Fondements techniques : comment Alibaba a construit son arme

Qwen 3.7-Plus n’est pas un simple upgrade. C’est une **réarchitecture complète** qui mélange :
- **Un mélange d’experts (MoE) optimisé** : 128 experts, mais avec un routing dynamique qui évite le gaspillage de calcul. Contrairement à Mixtral, où les experts sont statiques, ici, le modèle choisit en temps réel ceux qui comptent. Résultat : **meilleure efficacité énergétique**, et ça, les data centers chinois adorent.
- **Un contexte étendu à 1M de tokens** : oui, vous avez bien lu. Pas 128k, pas 256k, **un million**. Pour les ingénieurs qui bossent sur du RAG ou de l’analyse de code long, c’est un game-changer. Enfin, si votre GPU tient le coup.
- **Multimodalité native** : texte, image, audio, vidéo. Pas besoin de plug-ins ou de modèles séparés. Le tout dans un seul checkpoint. **GPT-4o, prends ça dans les dents.**

Le plus marrant ? Alibaba a entraîné ce modèle sur **des puces chinoises**. Pas de H100, pas de A100, mais des **Ascend 910B** de Huawei. ### L’architecture en détail (pour les puristes)

Qwen 3.7-Plus utilise une variante de **Transformer avec attention groupée (GQA)** pour réduire la latence. Le routing des experts est géré par un **gating network léger**, entraîné en parallèle avec le reste du modèle. Le tout est optimisé pour tourner sur **MindSpore**, le framework maison d’Huawei.

```python
# Exemple simplifié de routing dynamique dans Qwen 3.7-Plus
import torch
import torch.nn.functional as F

class DynamicRouter(torch.nn.Module):
    def __init__(self, num_experts, d_model):
        super().__init__()
        self.gate = torch.nn.Linear(d_model, num_experts)

    def forward(self, x):
        # x shape: [batch, seq_len, d_model]
        logits = self.gate(x)  # [batch, seq_len, num_experts]
        weights = F.softmax(logits, dim=-1)
        # Top-2 gating (comme dans Switch Transformer)
        top_k_weights, top_k_indices = torch.topk(weights, k=2, dim=-1)
        return top_k_weights, top_k_indices
```

Franchement, c’est propre. Et ça explique pourquoi le modèle tient la charge sur des benchmarks comme **MMLU-Pro** ou **GPQA Diamond**.

## Implémentation : comment l’utiliser (sans se ruiner)

Bon, parlons pratique. Qwen 3.7-Plus est **open-weight**, mais pas open-source au sens strict : Alibaba garde le code d’entraînement sous le coude. Dommage, mais c’est déjà mieux que rien.

### Déploiement local (si vous avez un data center dans votre garage)

Pour les chanceux qui ont **8x A100 80GB** (ou l’équivalent chinois), voici comment le faire tourner :


Alibaba pousse son modèle via **ModelScope**, sa plateforme open-source. L’API est simple, mais attention aux coûts :
- **Latence** : avec 1M de tokens de contexte, même un MoE bien optimisé met du temps.
- **Mémoire** : 300B params, c’est **600GB en float16**. Prévoyez du swap ou un bon café en attendant que ça charge.

## Limitations : les couacs qui font tiquer

1. **La multimodalité n’est pas parfaite** : Oui, il gère les images, mais essayez de lui faire analyser un schéma technique complexe. **Il hallucine des détails**. Comme tous les autres, d’ailleurs.
2. **Le fine-tuning est un casse-tête** : Avec 300B params, même un LoRA devient un projet de plusieurs jours. Et sans accès au code d’entraînement, optimiser le modèle est **un sport extrême**.
3. **La censure chinoise** : Qwen 3.7-Plus a été entraîné avec des **filtres politiques**. Si vous lui demandez des infos sur Taïwan ou le Tibet, préparez-vous à des réponses… créatives.
4. **Les coûts d’inférence** : Même avec un MoE, 300B params, ça coûte cher. **0.5 par million de tokens** en cloud (sur ModelScope), c’est loin d’être donné.
