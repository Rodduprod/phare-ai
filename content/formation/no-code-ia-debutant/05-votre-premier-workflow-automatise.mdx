## Le workflow qu'on va construire

**Objectif** : Chaque nouveau formulaire de contact sur votre site est automatiquement analysé par l'IA, classifié par type et urgence, et envoyé dans le bon canal selon sa priorité.

**Outils** : Typeform (ou Google Forms) + Make + ChatGPT + Slack/Email

## Les étapes

**Déclencheur** : Nouveau formulaire soumis sur Typeform
→ Make reçoit les données (nom, email, message)

**Étape IA** : Make envoie le message à l'API ChatGPT avec ce prompt :
```
Analyse ce message de contact.
Retourne un JSON avec :
- type: "commercial" | "support" | "partenariat" | "autre"
- urgence: "haute" | "normale" | "basse"
- résumé: [2 phrases max]
```

**Routage** :
- Si urgence = haute → Notification Slack immédiate dans `#urgent`
- Si type = commercial → Email à l'équipe commerciale
- Si type = support → Créer un ticket dans votre outil support
- Autres → Ajouter dans Notion pour traitement quotidien

**Confirmation** : Email automatique au contact : "Nous avons bien reçu votre message et reviendrons vers vous sous 24h."

## Construire étape par étape

Ne construisez pas tout d'un coup. Commencez par : déclencheur → étape IA → log dans Sheets. Vérifiez que l'IA classe correctement. Puis ajoutez le routage. Puis les notifications. Chaque bloc doit fonctionner avant d'ajouter le suivant.

## La documentation — obligatoire

Documentez votre workflow dans Notion ou un simple doc : déclencheur, chaque étape, données transmises, cas d'erreur connus. Dans 6 mois, vous ne vous souviendrez plus de rien.

## Ce qu'il faut retenir

Construisez bloc par bloc et testez chaque étape. L'IA dans le workflow prend des décisions intelligentes plutôt que de suivre des règles fixes. Documentez tout — votre futur vous remerciera. Et gardez une validation humaine pour les décisions importantes.
